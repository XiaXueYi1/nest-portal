# 项目基础设计与开发规范

## 1. 项目定位

- 技术栈：NestJS + TypeScript + Prisma + PostgreSQL + Redis
- 核心模块：
  - `auth`：登录、续签、登出、会话校验
  - `llm`：对话能力（普通响应 + SSE）
  - `common`：全局 guard/pipe/interceptor/filter/logger/prisma/redis

## 2. 目录约定

- `src/common/*`：全局与通用能力
- `src/modules/*`：业务模块（高内聚、低耦合）
- `src/config/*`：配置加载与校验
- `src/setup/*`：App 装配与启动初始化
- `prisma/*`：数据库 schema、迁移、生成配置
- `dosc/*`：设计文档与功能记录

## 3. 请求生命周期（当前实现）

1. 全局 `APP_GUARD`（默认鉴权，`@Public()` 跳过）
2. 全局 `APP_PIPE`（参数校验与转换）
3. 全局 `APP_INTERCEPTOR`（请求日志、响应包装）
4. Controller / Service
5. 全局 `APP_FILTER`（异常统一处理）

## 4. 认证规范

- 使用 `@nestjs/jwt` + `@nestjs/passport` + `passport-jwt`
- 双 token（access + refresh）通过 HttpOnly Cookie 传输
- Redis 存会话（sid），支持单会话退出 + 一键全下线
- `GET /auth/status` 用于前端登录态探活，并支持临期自动续签

## 5. 日志规范

- 使用 `nest-winston` + `winston`
- 等级：`error` / `warn` / `info` / `debug`
- 全局记录请求方法、路径、状态码、耗时、IP
- 异常统一在过滤器记录
- 默认脱敏敏感字段（password/token/cookie/authorization/secret）

## 6. 数据库规范

- Prisma + PostgreSQL
- schema 采用目录模式：`prisma/schema/schema.prisma + modules/*.prisma`
- 主键统一 `String @id @default(cuid())`
- 禁止物理外键，使用 `relationMode = "prisma"`
- 关系动作统一用 `Restrict`/`SetNull`，不使用 `NoAction`

## 7. 配置规范

- `ConfigModule` + `Joi` 统一校验
- 根据 `NODE_ENV` 读取 `.env.<env>`，开发默认回退 `.env.dev`
- 关键配置分组：`AUTH_*`、`REDIS_*`、`LOG_*`、`DATABASE_URL`、`DEEPSEEK_*`

## 8. 开发规范

- 使用 `@/` 路径别名（指向 `src/`）
- 代码格式：Prettier；代码规范：ESLint
- 方法使用多行 `@description` 注释；核心类型字段添加中文注释
- 新增模块必须提供模块内文档，并同步更新 `dosc/02-功能完成记录.md`
