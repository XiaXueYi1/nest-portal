# 数据库处理规范（Prisma + PostgreSQL）

## 1. 目标

- 采用 RBAC 模型（用户、角色、权限、关联关系）
- 主键统一字符串 ID（`cuid()`），禁止自增 ID
- 禁止物理外键，使用 Prisma 关系层（`relationMode = "prisma"`）

## 2. Schema 组织

- 主入口：`prisma/schema/schema.prisma`（只放 generator + datasource）
- 模块目录：`prisma/schema/modules/`
  - `user.prisma`
  - `user_credential.prisma`
  - `role.prisma`
  - `permission.prisma`
  - `permission_control_relations.prisma`
  - `auth_session.prisma`
  - `audit_log.prisma`

## 3. 命名约定

- 表名：复数下划线（如 `user_roles`）
- 字段名：小驼峰，通过 `@@map` 映射表名
- 关联表必须设置联合唯一索引（如 `@@unique([userId, roleId])`）

## 4. 关系与约束

- `relationMode = "prisma"` 时，不使用 `NoAction`
- 关系动作优先使用：
  - `Restrict`
  - `SetNull`（仅可空关系字段）
- 删除策略由应用层显式实现，不依赖数据库级联

## 5. 索引要求

- 高频查询字段必须建索引
- RBAC 关键表最少索引要求：
  - `user_roles(userId, roleId)`
  - `role_permissions(roleId, permissionId)`
  - `auth_sessions(userId, refreshExpiresAt)`
  - `audit_logs(actorUserId, action, createdAt)`

## 6. 迁移与生成

- 生成客户端：`pnpm run prisma:generate`
- 开发迁移：`pnpm run prisma:migrate:dev -- --name <migration_name>`
- 生产迁移：`pnpm run prisma:migrate:deploy`
- 可视化：`pnpm run prisma:studio`

## 7. 变更流程

1. 修改 `prisma/schema/modules/*.prisma`
2. 先执行 `prisma validate`
3. 再执行 `prisma migrate dev`
4. 生成并提交迁移 SQL 与文档更新
